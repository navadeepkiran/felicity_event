# Felicity Event Management System - Deployment Links

## Production URLs

### Frontend URL
https://felicity-event.vercel.app

### Backend API URL
https://felicity-event.onrender.com

## Deployment Details

### Frontend Hosting
Provider: Vercel / Netlify
Build Command: npm run build
Output Directory: dist

### Backend Hosting
Provider: Render / Railway / Heroku
Build Command: npm install
Start Command: npm start
Port: Environment Variable

### Database
Provider: MongoDB Atlas
Connection: Via MONGODB_URI environment variable

## Environment Variables (Backend)

Required environment variables set in hosting platform:
- PORT
- NODE_ENV=production
- MONGODB_URI
- JWT_SECRET
- ADMIN_EMAIL
- ADMIN_PASSWORD
- EMAIL_HOST
- EMAIL_PORT
- EMAIL_USER
- EMAIL_PASSWORD
- FRONTEND_URL

Backend health endpoint: https://felicity-event.onrender.com/api/health
Expected response: {"status":"ok","message":"Server is running"}

## Known Limitations

- **Email Delivery:** Gmail SMTP experiences connection timeouts from Render servers. For production email delivery, integrate SendGrid or Mailgun. Users can access QR codes directly from their dashboard.
- **Free Tier Limitations:** 
  - Render backend may spin down after 15 minutes of inactivity (30s cold start)
  - MongoDB Atlas limited to 512MB storage
  - Vercel bandwidth limits apply
